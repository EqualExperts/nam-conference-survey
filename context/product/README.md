# Context - Product Management Documentation

This directory contains context files for product management workflows.

## Shared Documents (Read by Both PM and Engineering)

- **product-spec.md** - What we're building (updated via /rel)
- **backlog.md** - What's left to build (managed via /req and /rel)

## Product Context

Product context for discovery synthesis and story extraction:

| File | Purpose |
|------|---------|
| product-overview.md | Business context and goals |
| target-users.md | User personas and needs |
| competitive-landscape.md | Market analysis |
| glossary.md | Domain terminology |
| technical-context.md | Technical constraints for PM |

**When to read**: During /synth, /req commands
**When to skip**: Pure engineering implementation tasks

---

## Usage

These files provide essential context for:
- Discovery synthesis
- Story extraction
- Requirements gathering
- Release planning

Slash commands like `/synth` and `/req` reference these files to understand the product domain.
